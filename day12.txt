Event handling is the process of detecting and responding to events (actions or occurrences) in a program. Events can be generated by a user (such as clicking a button or pressing a key) or by the system (such as a timer expiring or a file finishing loading).

Common Events
Mouse click
Keyboard key press
Mouse movement
Form submission
Window resize
Timer completion
How Event Handling Works
An event occurs.
The program detects the event.
An event handler (a function or method) is executed in response.
Example (JavaScript)
<button id="btn">Click Me</button>

<script>
document.getElementById("btn").addEventListener("click", function() {
    alert("Button clicked!");
});
</script>

Explanation:

addEventListener() registers an event handler.
"click" specifies the event type.
The function runs whenever the button is clicked.
Components of Event Handling
Event Source: The object that generates the event (e.g., a button).
Event: The action that occurs (e.g., click, keypress).
Event Handler (Listener): The code that responds to the event.
Advantages
Makes applications interactive.
Improves user experience.
Separates event detection from business logic.
Enables responsive graphical user interfaces (GUIs).
Applications
Web development (HTML, CSS, JavaScript)
Desktop GUI applications (Java Swing, JavaFX, .NET)
Mobile apps (Android, iOS)
Game development

In simple terms: Event handling is the mechanism that allows a program to respond to user actions or system-generated events by executing specific code.
